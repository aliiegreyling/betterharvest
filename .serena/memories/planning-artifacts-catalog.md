# Planning artifacts catalog

Map of every BMAD planning artifact in this repo. All under `_bmad-output/planning-artifacts/`.

## Foundational

### `agentic-build-system-plan.md`
The original full SDLC plan for Forge. Sections 1–9: vision, SDLC approach, product brief, PRD, UX (CLI), architecture, epics & stories, implementation readiness, sprint plan.

Key claims:
- Goal: ≥40% cost reduction vs. always-Opus.
- Reference set of ≥5 prompts of varied complexity must produce runnable projects.
- Medium-prompt run < 10 min wall-clock.
- Structured audit log every run.

## Agentic Workflow CLI Harness brief (2026-05-21)

Folder: `agent-cli-harness-brief-2026-05-21/`.

### `brief.md`
Product brief for the Agentic Workflow CLI Harness. Covers problem (greenfield fragmentation + brownfield amnesia), solution (BMAD spine + Serena intelligence + MCP-first extensibility + per-step model routing), Forge v0.1 baseline, success criteria, scope, platform direction, early workflow shape (greenfield + brownfield example flows), open questions, vision.

### `addendum.md`
Source input + notes for PRD + **Forge v0.1 baseline analysis** (what exists, gap analysis vs. brief, recommended product direction layered into v0.2 then v0.5+, candidate v0.2 requirements, **v0.2 implementation progress** showing what was shipped, product decisions implied by current state).

The v0.2 implementation progress section is the closest thing to a current-state changelog for the harness slice.

### `.decision-log.md`
Chronological log of decisions made during planning. Latest: TypeScript/Node + CLI-only + Forge as accepted baseline decisions for next PRD pass. v0.2 foundation slice implemented + verified with typecheck/build/smoke tests.
